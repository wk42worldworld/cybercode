# CyberCode v1.0.19

## Fixes

- Split Kimi Code and Kimi API model presets so Kimi K2.7 Code models stay on the Kimi Code endpoint, while the Kimi API endpoint defaults to Kimi K2.6.
- Added provider migration and runtime guards for stale model IDs, including `mimo-v2.5-flash`, `glm-4.5-air`, and misplaced Kimi K2.7 Code models.
- Improved third-party provider thinking handling so MiMo and similar models no longer fail when an internal side query tries to disable thinking.
- Reworked image capability handling with Auto / Enabled / Disabled provider settings, dynamic model inference, and legacy config migration.
- Let non-vision models receive image file paths for OCR/MCP tool use instead of failing before tools can run.
- Improved queued user-message handling while the assistant is busy, including merged queued message display and automatic sending after the current turn.
- Updated desktop sidebar behavior and provider logo rendering for better consistency.
