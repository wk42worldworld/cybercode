# CyberCode v1.0.9

## Desktop

- Added a running-turn input buffer: when the assistant is already responding, new input appears in a pending bar where it can be edited, deleted, added to the current task, or sent after the current turn.
- Added WebSocket steering and cancellation status events so queued input stays visible and recoverable while the model is working.
- Improved provider/model context-window handling for desktop sessions, including preset metadata and runtime forwarding for long-context models.
- Improved Windows shell fallback behavior by using Git Bash when available, falling back to PowerShell when needed, and hiding unavailable Bash tools from the model.
- Improved command and chat UI polish, including active command highlighting, tighter message spacing, and a safer virtualized message scroller.

## Reliability

- Made scheduled-task execution timeout configurable for tests while preserving the production default.
- Stabilized E2E WebSocket tests by using the mock SDK CLI and current model catalog expectations.
- Updated the GitHub homepage README highlights for the latest desktop behavior.
