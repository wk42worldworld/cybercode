## Highlights

- Added provider-aware image-input capability handling so text-only models no longer receive image blocks they cannot process.
- Added a provider settings switch for image input, with model-name inference for common vision and text-only models.
- Changed desktop updates to predownload silently in the background and only show the smaller blue rail update button once the update is ready to install.

## Fixes

- Block image attachments before starting a text-only desktop chat session, preventing unsupported multimodal requests from poisoning the conversation.
- Replace agent/tool-generated image blocks with a text-only placeholder when the selected model cannot read images, so the model can respond normally instead of repeating an API error.
- Recover media-tainted sessions after request-size or unsupported-image errors by stopping the bad run and trimming the latest media turn.
- Keep runtime model image capabilities aligned when a session switches models inside the same provider.
- Removed the global update popup; update install is now initiated from the left rail after the download is complete.
