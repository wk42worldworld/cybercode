# CyberCode v1.0.21

## Highlights

- Added provider-independent web search for every supported model, with Brave, Bing, DuckDuckGo, and Baidu failover, domain filtering, result deduplication, caching, and fallback from unavailable native search.
- Added a live GitHub Trending adapter that reads the official daily, weekly, or monthly page directly and returns ranked, timestamped, structured repository data instead of stale search summaries.
- Added provider model discovery for OpenAI-compatible endpoints, Ollama, and LM Studio. Discovered model IDs, context windows, and image capabilities now flow into the desktop model selector.
- Expanded provider diagnostics to test every configured role model and optionally probe image input support, with per-model results shown in Desktop settings.
- Added learned image-capability caching and automatic recovery when a provider rejects image input, retrying the turn as a file reference so OCR and MCP tools can still process it.

## Fixes

- Prevented third-party Anthropic-compatible gateways such as Kimi and GLM from receiving unsupported `tool_reference` blocks unless compatibility is explicitly enabled.
- Corrected endpoint-aware thinking policies for Kimi Code, Zhipu GLM, and Xiaomi MiMo, and aligned provider presets and context-window handling with the actual selected model.
- Fixed parallel and historical tool cards remaining permanently animated after completion, interruption, restart, or reconnect; interrupted calls now show a stopped state.
- Kept the desktop chat state in tool execution mode until every pending tool in the latest turn has returned.
- Hardened image fallback, runtime provider switching, and session restart behavior so failed media turns do not terminate an otherwise usable conversation.

## Security And Reliability

- Protected the desktop local server with a per-launch random token and forwarded the same authorization to REST, WebSocket, Feishu, and Telegram adapter connections.
- Masked credentials returned by settings APIs, preserved existing secrets when masked values are saved back, and restricted local settings and provider files to owner-only permissions.
- Improved provider URL normalization, masked-key handling, model migration, and conservative context-window resolution for shared role models.
