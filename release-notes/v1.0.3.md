# CyberCode v1.0.3

## Release Experience

- Restored Tauri updater artifact generation for desktop releases.
- Added the updater signing key used by GitHub Actions so release builds can publish signed updater bundles and `latest.json`.
- Rotated the updater public key in the desktop app; users on v1.0.2 should install this release manually once, then future updates can use the built-in updater.
- Added a release preflight check that fails clearly if the updater signing secret is missing.
- Added default macOS ad-hoc signing when Apple notarization is not explicitly enabled, reducing the chance that downloaded macOS builds are treated as damaged.
- Guarded Apple notarization behind `ENABLE_APPLE_NOTARIZATION=true` so stale Apple credentials no longer break normal release builds.

## Verification

- Validated workflow YAML and Tauri JSON configuration.
- Confirmed the committed updater public key matches the locally backed-up private key.
- Ran a local macOS ARM app bundle build with ad-hoc signing and confirmed `CyberCode.app.tar.gz.sig` is generated.
