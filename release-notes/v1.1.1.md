# CyberCode v1.1.1

## Highlights

- Added native region screenshot capture from the chat composer, with a familiar drag, move, resize, confirm, and cancel workflow.
- Stabilized assistant streaming so already revealed characters no longer change and completed replies no longer cause a final bubble-size or scroll jump.
- Improved provider and process failure handling so billing exhaustion and unexpected CLI exits produce useful recovery behavior instead of a misleading hang.

## Chat Experience

- Keeps live text reveal strictly ordered while preserving the model's exact final response and Markdown rendering.
- Prevents transient invalid virtual-list measurements and negative history indexes that could cause conversation scroll shake.
- Aligns chat bubbles, execution blocks, task bars, pending guidance, loading notices, and team status content to the same readable content column.
- Refines assistant bubble corners, tool-block spacing, short-message wrapping, and minimum-window layout behavior.
- Keeps token controls, model selection, and send controls readable at the supported `960 x 640` minimum window size.

## Screenshot Capture

- Adds a scissors button to the composer for selecting and attaching a screen region.
- Supports native capture entry points on macOS, Windows, and Linux, with platform fallbacks where required.
- Adds the macOS screen-capture usage description and the Tauri permissions required by the overlay window.

## Reliability And Permissions

- Automatically approves tool permission requests in `bypassPermissions` mode while still showing questions that genuinely require user input.
- Retries one recoverable send when the CLI process exits between prewarm and message delivery.
- Recognizes GLM and other OpenAI-compatible provider balance or quota exhaustion as a permanent billing error, avoids pointless retries, and shows a recharge/quota message.
- Improves long-running response status and preserves actionable raw provider details in error displays.

## Distribution

- Cloud builds cover macOS Apple Silicon, macOS Intel, Linux x64, and Windows x64.
- Every platform includes a ZIP download in addition to its normal DMG, DEB, or NSIS installer and updater artifacts.
- Updater manifests retain the direct GitHub URL and existing mirror fallbacks.

- Passed all 12 sidecar tests and all 10 Tauri Rust tests.
- Passed TypeScript checking, the desktop production build, Rust formatting, macOS ARM64 and Intel Rust checks, configuration validation, and a real-browser minimum-window smoke test with zero console errors or warnings.
