# CyberCode v1.0.23

## Highlights

- Added a native Token Optimization workspace with independent global switches, consistent full-cycle savings estimates, and an animated minimum/maximum range overview.
- Integrated CodeGraph directly into CyberCode. Projects are indexed automatically after the global switch is enabled, newly written code is refreshed without extra setup, and the graph can be inspected in an interactive visualization.
- Embedded RTK 0.43.0 for macOS, Windows, and Linux so Git, test, build, terminal, and Docker output can be compressed without a separate installation.
- Added Agent Data Migration for compatible skills, memories, instructions, and project context across CyberCode, Claude Code, Codex, Cursor, OpenClaw, Hermes Agent, and DeepSeek TUI / CodeWhale.

## Token Optimization

- Added CodeGraph symbol and relationship lookup rules so the agent can inspect architecture and impact before reading full files.
- Added RTK tool-output compression, Caveman response compression, Lite deterministic prompt cleanup, and the user-facing Lazy Programmer minimal-implementation mode.
- Added automatic CodeGraph refresh for existing projects, cloned repositories, empty projects that later receive code, and files created by the agent.
- Added WAL checkpointing and index maintenance to keep CodeGraph storage bounded during repeated indexing.
- Added bundled grammar support for TypeScript, JavaScript, Python, Go, Rust, Java, C, PHP, Lua, Solidity, TSX, and JavaScript embedded in HTML.

## Desktop Experience

- Added branching from any completed assistant reply. A branch preserves the conversation up to the selected response, opens independently, and leaves the source conversation unchanged.
- Added project display-name and session-title editing from the sidebar, including persistent aliases, cancel behavior, and project-aware tab updates.
- Removed visible project paths from sidebar rows while keeping the full path available on hover.
- Refined assistant message actions with clearer spacing and alignment.
- Improved token usage accounting and hover details for live turns, cached usage, and restored sessions.
- Reworked the startup surface so the branded splash appears immediately without the previous blank, yellow, or white flash.
- Added unified provider and agent artwork while keeping the compact composer model selector text-only.

## Reliability And Compatibility

- Hardened Windows sidecar builds with a broadly compatible Bun baseline target and clearer startup diagnostics for unsupported CPUs or damaged runtimes.
- Added sidecar startup smoke tests and cross-platform helper tests to the four-target Release workflow.
- Fixed built-in tool-result mapping so SkillGate and other tools always return a valid model-facing result contract.
- Fixed local model image detection so Ollama and LM Studio runtime metadata overrides optimistic preset guesses.
- Kept existing provider models, token-optimization settings, project paths, and session data compatible during upgrade.

## Distribution

- Cloud builds cover macOS Apple Silicon, macOS Intel, Linux x64, and Windows x64.
- Every platform includes a standalone ZIP download in addition to its normal DMG, DEB, or NSIS installer and updater artifacts.
- macOS builds use the configured signing and notarization pipeline; updater manifests include the existing GitHub mirror fallbacks.
