# CyberCode v1.0.5

## Memory

- Added the first complete Skill Memory layer: invoked skills can now keep compact learned notes in per-skill `SUMMARY.md` files and reuse them on future calls.
- Added automatic Skill Memory review so repeated durable lessons are summarized after enough observations instead of growing the active prompt.
- Added `SkillGate` to check proposed new skills against existing skills before creation, helping avoid duplicate skill sprawl.
- Added `SkillMemory` governance actions for reading skill memory, pinning or archiving skills, scanning duplicate clusters, and safely merging duplicate skill summaries.
- Added low-frequency lifecycle management: unused skills can be marked `stale` or `archived` without deleting files or editing `SKILL.md`.

## Safety

- Archived skill memory is no longer injected automatically, while the underlying skill remains available for manual use.
- Duplicate skill merging is limited to `SUMMARY.md` memory notes and does not rewrite skill source files.
- Governance refreshes run as lightweight background maintenance and avoid automatic deletion or skill-file mutation.

## Verification

- Ran focused Skill Memory, Prompt Memory, Session Search, and Skills API regression tests.
- Ran import-chain checks for the new Skill Memory governance tools.
