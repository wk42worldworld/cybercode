# CyberCode v1.0.2

## Memory System

- Added the first-layer prompt memory system with compact `SOUL.md`, `BRIEF.md`, and `USER.md` files.
- Added automatic prompt-memory review so stable user preferences and project facts can be collected over time.
- Added the second-layer session search memory powered by bundled SQLite FTS5/trigram indexing.
- Added a read-only `SessionSearch` tool so the agent can search and reopen useful historical conversation context.
- Indexed current sessions at turn end for both CLI and desktop, with search-time refresh as a fallback.

## Configuration

- Moved CyberCode-owned configuration defaults to `~/.cyber`.
- Renamed CyberCode project guidance files toward `CYBER.md` / `cyber.md` naming while preserving compatibility paths where needed.

## Desktop

- Added prompt-memory API wiring for the desktop client.
- Improved session search integration used by the desktop search surface.
- Kept desktop release targets on macOS Apple Silicon, macOS Intel, Linux x64, and Windows x64.
- macOS packages are currently distributed without Apple notarization; if Gatekeeper blocks launch, run `sudo xattr -cr /Applications/CyberCode.app` after installing.

## Documentation

- Refreshed the GitHub homepage branding and removed stale logo assets.
- Added prominent language entry points with English as the default project homepage and Chinese, Japanese, and Korean variants.

## Verification

- Ran the focused session-search and server regression suite.
- Ran desktop TypeScript checks and production web build.
