# CyberCode v1.0.22

## Highlights

- Added a self-evolution memory dashboard that organizes what CyberCode knows about the user and its reusable working methods, with categorized insights that can be reviewed, edited, or removed.
- Added Skill Learning with `Off`, `Suggest`, and `Auto` modes. Complex completed tasks can become reusable project or global skills, with review history, approval controls, duplicate protection, backups, and secret redaction.
- Added guided provider setup to the terminal TUI and a built-in protocol bridge for Anthropic-compatible, OpenAI Chat, and OpenAI Responses providers, so normal provider setup no longer requires a separately installed proxy.
- Added a standalone ZIP download for every cloud-built desktop target: macOS Apple Silicon, macOS Intel, Linux x64, and Windows x64.

## Desktop Experience

- Added live token and context-window usage in chat, backed by session usage and transcript estimates.
- Expanded local slash-command panels and made streaming status text smoother and fully localized in Chinese, English, Japanese, and Korean.
- Improved drag-and-drop attachments and preserved unsupported file types as usable file references instead of blocking a model request.
- Improved temporary-session discovery and first-turn project memory, plus direct access to memory from the chat rail.
- Refined modal sizing, settings navigation, pending chat state, and low-resolution layout behavior.

## Models And Providers

- Updated provider model choices and official context-window defaults for GLM, Kimi, MiniMax, OpenAI, LM Studio, and Ollama.
- Corrected Kimi API routing, thinking parameters, and reasoning-content continuity for tool-call conversations.
- Added conservative migrations for known stale preset values while preserving custom provider models and existing user configuration.
- Changed new desktop sessions to use the configured bypass-permissions default consistently.

## Distribution And Documentation

- Added GitHub updater mirror manifests and public mirror fallback endpoints for regions where direct GitHub downloads are unreliable.
- Improved the Windows CLI bootstrap and PATH handling, including selecting one available runtime command.
- Expanded the multilingual documentation with guided provider setup, desktop/TUI/shell command distinctions, slash-command references, release mirrors, and the official website link.
- Removed accidentally tracked terminal screen logs and excluded future `screenlog.*` files from source packages.
