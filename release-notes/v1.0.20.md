# CyberCode v1.0.20

## Improvements

- Reworked the desktop sidebar session list to group conversations by project, keep temporary sessions together, support project collapse, and allow removing a project from the sidebar without deleting files.
- Added a dedicated new-project dialog so users can create a project folder from the sidebar with a native folder picker instead of typing paths manually.
- Changed project filtering to single-select mode with explicit All Projects and Temporary Sessions choices, and kept new-chat creation from unexpectedly changing the active filter.
- Improved desktop image attachments selected through the native file picker: local previews now render as data URLs, while the backend still receives the original local path for model and MCP tool use.
- Kept the chat composer clickable above the bottom overlay and preserved local image previews in sent messages and queued follow-up messages.
- Hardened provider/model configuration for Kimi, Kimi Code, Zhipu GLM, MiMo, and custom Anthropic-compatible providers, including enabled-thinking policy defaults, stale model repair, current-provider model validation, and clearer provider API errors.
