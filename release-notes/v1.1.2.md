# CyberCode v1.1.2

## Highlights

- Makes local conversation switching feel immediate with background prefetch, shared in-flight reads, stale-path recovery, and no model runtime startup while browsing history.
- Expands Token Optimization with a global Smart Pruning switch, adjustable pruning strength, stricter on/off behavior, and improved Code Graph context selection.
- Improves long-term memory and self-evolution with lightweight project summaries, searchable cross-session recall, six-message periodic review, and explicit memory-injection controls.
- Refreshes provider setup with recent model presets plus custom model IDs, corrected context-window defaults, image-capability handling, and Kimi K3 options without overwriting existing user configuration.

## Desktop Experience

- Keeps streamed reply text in strict reveal order and prevents already displayed characters from changing.
- Reduces final-message bubble resizing, scroll jumps, short-message wrapping errors, and inconsistent content widths.
- Adds native region screenshot capture and drag-and-drop attachments from the composer.
- Handles unsupported attachments as local file references so an incompatible file type cannot stall the model request.
- Improves command execution spacing, running-state highlights, question forms, permission handling, and low-resolution layouts.

## Code Graph And Token Optimization

- Automatically prepares project-scoped graph context for both newly created projects and existing repositories when the global graph switch is enabled.
- Prioritizes symbol, relationship, architecture, and impact lookup before reading large source files.
- Improves graph freshness after files are created or changed and provides a denser interactive visualization.
- Ensures Smart Pruning, Lite cleanup, RTK optimization, Code Graph, and other optimization switches apply globally when enabled and stop applying when disabled.
- Keeps conservative defaults for aggressive prompt strategies while preserving user-selected optimization settings.

## Memory, Models, And Migration

- Stores compact project-path and work summaries from temporary conversations for later FTS-backed recall.
- Adds a user-facing switch for injecting learned evolution memory into future tasks.
- Supports preset model dropdowns and free-form model entry together, ordered from newer to older options.
- Improves source-to-destination Agent migration for skills, memories, project instructions, and individually selected files.
- Preserves existing provider credentials, model mappings, context limits, and migration destinations during upgrades.

## Reliability And Privacy

- Recovers local history by session ID when a project was moved or an older stored path is no longer valid.
- Distinguishes billing, quota, process-exit, and local-history failures instead of presenting a generic stalled response.
- Prevents automatic composer focus from launching CLI or Code Graph processes; runtime prewarming now requires real typing, paste, or pointer intent.
- Adds a best-effort anonymous daily activity heartbeat containing only a hashed installation ID, app version, platform, architecture, and surface. It honors the existing analytics opt-out, `DO_NOT_TRACK`, and `CYBERCODE_USAGE_ANALYTICS_DISABLED`, and never blocks startup or chat.

## Distribution

- Cloud builds cover macOS Apple Silicon, macOS Intel, Linux x64, and Windows x64.
- Every platform includes a ZIP download alongside its normal DMG, DEB, or NSIS installer and updater artifacts.
- Updater manifests retain direct GitHub downloads and the configured mirror fallbacks.

## Verification

- Validated desktop UI, local-history, provider, memory, optimization, Code Graph, sidecar, server/API, and core behavior with automated regression suites.
- Passed desktop TypeScript checks, production web compilation, documentation compilation, sidecar smoke checks, and release configuration validation.
