# CyberCode v1.0.6

## Desktop Reliability

- Added Windows shell fallback behavior so CyberCode uses Git Bash when available, falls back to PowerShell when it is not, and avoids exposing unavailable Bash tools to models.
- Improved unsupported attachment handling by sending desktop-selected files as filesystem paths instead of forcing inline payloads, preventing large audio or binary files from freezing model turns.
- Added server-side attachment policy checks so path-only file attachments are accepted while unsupported inline regular files fail fast with a clear error.
- Added long-running turn activity tracking and UI notice behavior for models that do not stream visible thinking, making it clearer when a session is still working.

## Chat UI

- Added a chat mode sidebar entry point for compact switching.
- Added GPT-style running text sweep animation to active tool execution labels, including parent execution rows while child commands are still running.
- Refined settings and localized copy for the new runtime and attachment states.

## Verification

- Ran root attachment and Windows shell regression tests.
- Ran the full desktop Vitest suite.
- Ran the desktop production build.
