## English

### Short Post

I am building CyberCode, an open-source Claude Code-style desktop agent that can connect to custom model providers.

It supports a Tauri desktop app, CLI/headless mode, provider presets, context-window metadata, Telegram/Feishu remote control, scheduled tasks, multi-agent workflows, Skills, MCP, and Computer Use on macOS/Windows.

Repo: https://github.com/wk42worldworld/cybercode

### Show HN / Reddit Style

I wanted a Claude Code-like workflow that I could run locally with my own model provider, plus a desktop app for day-to-day sessions.

CyberCode is my open-source attempt at that. It includes a terminal UI, a Tauri desktop app, custom provider configuration, cross-platform desktop builds, Telegram/Feishu remote control, scheduled tasks, memory, multi-agent workflows, Skills, MCP, and Computer Use.

I would love feedback on the first-run experience, provider setup, and which model/provider combinations work best for you.

## Posting Checklist

- Lead with one use case, not the full feature list.
- Include one screenshot, GIF, or short screen recording when possible.
- Ask for specific feedback: provider setup, desktop install, first-run UX, or one model combination.
- Link directly to the release page when the audience wants to install, and to the repository when they want to inspect the code.
