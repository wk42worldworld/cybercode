# Knowledge Space와 Code Graph

CyberCode는 프로젝트 코드 구조와 사용자가 관리하는 로컬 자료를 하나의 Knowledge Space에 모읍니다. 데스크톱 오른쪽 레일의 그래프 아이콘을 누르면 Token Optimization을 거치지 않고 바로 열립니다.

![CyberCode Code Graph](../../images/gui/cybercode-code-graph.png)

## 두 가지 인덱스

### Code Graph

- **대상:** 파일, 클래스, 함수, 메서드, 참조, 호출, 포함 관계.
- **주요 용도:** 구현 위치 탐색, 아키텍처, 영향도 분석, 다중 파일 리팩터링.
- **모델 사용:** 적합한 코딩 작업에 압축된 그래프 컨텍스트를 자동 제공하고 Agent가 그래프 도구를 사용합니다.

### Knowledge sources

- **대상:** 사용자가 선택한 로컬 파일 / 폴더.
- **주요 용도:** 자료 정리, 파일 목록, 로컬 전문 검색.
- **모델 사용:** 현재는 Knowledge Space 내 사용자 검색용이며, 자료 전체를 매 요청마다 모델에 넣지 않습니다.

## Code Graph 켜기

1. CyberCode에서 구체적인 프로젝트 폴더를 엽니다. 파일 시스템 루트나 홈 디렉터리 전체는 선택하지 마세요.
2. 오른쪽 레일의 그래프 아이콘을 누릅니다.
3. **Graph** 탭에서 **Enable Code Graph**를 누릅니다.
4. 상태가 **Ready**가 될 때까지 기다립니다.

이 작업은 전역 스위치를 켜며, 이후에 여는 프로젝트도 필요할 때 자동으로 인덱싱합니다. 전역으로 끄려면 **Token Optimization**을 사용하세요.

## Agent가 그래프를 사용하는 방법

- 요청과 관련된 심볼을 찾고 호출, 참조, 포함 관계를 따라 이웃 노드를 확장합니다.
- 결과의 순위를 정해 약 1,800 Token 이하의 압축 컨텍스트를 만듭니다.
- 넓은 범위의 파일을 읽기 전에 Agent에 제공하고, 더 깊은 그래프 조회 도구도 제공합니다.
- 일반 대화, 매우 짧은 입력, 명령 전용 요청은 그래프 컨텍스트를 추가하지 않습니다.

첫 인덱스 이후에는 백그라운드 감시기가 프로젝트 변경을 따라갑니다. 빈 프로젝트에서 새 코드를 만들어도 자동으로 갱신됩니다. 대규모 생성이나 이름 변경 직후에는 **Rebuild Index**를 사용하세요.

## 시각화 사용

- 심볼, 파일, 모듈을 검색하고 Enter를 눌러 첫 번째 일치 항목에 포커스합니다.
- **Architecture**는 의미 커뮤니티와 중요 노드를, **Files**는 파일별 구조를 보여 줍니다.
- 드래그로 이동하고 마우스 휠 또는 버튼으로 확대/축소합니다.
- 노드를 선택하면 소스 위치, 종류, 인덱싱된 관계를 볼 수 있습니다.

## 로컬 Knowledge sources 추가

**Sources** 위의 파일/폴더 추가 버튼을 사용하거나 Knowledge Space에 드래그 앤 드롭합니다. **Files**에서 인덱스 방식을 확인하고, **Search**에서 모든 소스 또는 선택한 소스를 검색합니다. 좁은 창에서 왼쪽 레일이 숨겨져도 추가, 재인덱싱, 제거 조작은 위쪽 도구 모음에 남아 있습니다.

`.git`, `node_modules`, `dist`, `build`, `target` 등은 건너뜁니다. 텍스트 파일은 전문을 인덱싱하고, 이미지·오디오·비디오·압축 파일은 파일명, 경로, 크기 같은 메타데이터만 인덱싱합니다. 소스를 제거해도 원본 파일은 삭제되지 않습니다.

## 지원 언어와 로컬 데이터

데스크톱 패키지에 필요한 파서와 MCP 브리지가 포함되어 추가 설정이 필요하지 않습니다. 현재 `HTML 내 JavaScript`, `TypeScript`, `TSX`, `JavaScript`, `Python`, `Go`, `Rust`, `Java`, `C`, `PHP`, `Lua`, `Solidity`를 지원합니다.

프로젝트 그래프는 `.codegraph/`에 저장되는 재생성 가능한 로컬 데이터이며 보통 Git에 커밋하지 않습니다. 잘못된 전체 스캔과 무제한 디스크 사용을 막기 위해 파일 시스템 루트나 홈 디렉터리 전체는 인덱싱할 수 없습니다.
