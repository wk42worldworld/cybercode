# スラッシュコマンドリファレンス

スラッシュコマンドは `/` で始まるコマンドです。Windows パスで使うバックスラッシュ `\` とは別物です。

## 3 つの入口

| 入口 | 使い方 | 説明 |
|------|--------|------|
| ターミナル TUI | `cybercode` を起動し、入力欄で `/` を入力 | セッション、コンテキスト、プラグイン、MCP、権限などを操作できる最も完全な対話型入口です。 |
| デスクトップ入力欄 | デスクトップアプリのチャット入力欄で `/` を入力 | Settings、Models、MCP、Skills、Memory などの UI ショートカットを優先して処理します。非対応の TUI コマンドはターミナル利用を案内します。 |
| 通常の shell | `cybercode <subcommand>` を実行 | スラッシュコマンドではありません。スクリプト、CI、`cybercode mcp list` のような管理操作に使います。 |

## ターミナル TUI の内蔵コマンド

| コマンド | 別名 / 引数 | 役割 |
|----------|-------------|------|
| `/help` | - | ヘルプと現在利用できるコマンドを表示します。 |
| `/status` | - | バージョン、モデル、アカウント、API 接続、ツール状態を表示します。 |
| `/add-dir` | `<path>` | 現在のセッションにアクセス可能な作業ディレクトリを追加します。 |
| `/context` | - | 現在のコンテキスト使用量を表示します。対話モードではグリッド表示になります。 |
| `/cost` | - | 現在のセッションの時間とコストを表示します。 |
| `/clear` | `/reset`, `/new` | 会話履歴を消去してコンテキストを空けます。 |
| `/compact` | `[要約指示]` | 会話履歴を圧縮し、要約を残して続行します。 |
| `/resume` | `/continue [会話 ID または検索語]` | 過去の会話を再開します。 |
| `/rename` | `[名前]` | 現在の会話名を変更します。 |
| `/branch` | `/fork [名前]` | 現在位置から会話ブランチを作成します。 |
| `/rewind` | `/checkpoint` | コードまたは会話を以前の地点へ戻します。 |
| `/exit` | `/quit` | REPL を終了します。 |
| `/copy` | `[N]` | 最新または N 番目に新しい AI 応答をコピーします。 |
| `/export` | `[ファイル名]` | 現在の会話をファイルまたはクリップボードへ書き出します。 |
| `/diff` | - | 未コミット変更とターンごとの差分を表示します。 |
| `/tasks` | `/bashes` | バックグラウンドタスクを表示、管理します。 |
| `/buddy` | `[hatch|pet|mute|unmute|info]` | コンパニオンやヒント系の対話機能を開きます。 |
| `/btw` | `<質問>` | メイン作業を止めずに短い横道質問をします。 |
| `/plan` | `[open または説明]` | プランモードを有効化するか、現在のプランを表示します。 |
| `/review` | `[PR 番号]` | Pull Request のレビューを Agent に依頼します。 |
| `/pr-comments` | `[PR 番号]` | GitHub Pull Request のコメントを取得し整理します。 |
| `/security-review` | - | 現在のブランチ変更をセキュリティ観点でレビューします。 |
| `/init` | - | プロジェクトの `CYBER.md` を初期化します。 |
| `/statusline` | `[指示]` | ステータスラインの表示を設定します。 |
| `/insights` | - | CyberCode セッションの分析レポートを生成します。 |
| `/model` | `[モデル]` | 現在のセッションで使うモデルを切り替えます。 |
| `/provider` | `/providers` | モデルプロバイダーを設定・切り替えます。OpenAI 互換 API は内蔵プロトコルブリッジを使用します。 |
| `/effort` | `low`, `medium`, `high`, `max`, `auto` | モデルの推論強度を調整します。 |
| `/fast` | `on`, `off` | 利用可能な場合に高速モードを切り替えます。 |
| `/config` | `/settings` | 設定パネルを開きます。 |
| `/permissions` | `/allowed-tools` | ツールの許可、拒否ルールを管理します。 |
| `/sandbox` | `exclude "command pattern"` | サンドボックスとコマンド除外ルールを設定します。 |
| `/theme` | - | ターミナルテーマを変更します。 |
| `/color` | `<色 または default>` | セッションのプロンプトバー色を設定します。 |
| `/vim` | - | Vim 編集モードと通常編集モードを切り替えます。 |
| `/terminal-setup` | - | ターミナルの改行ショートカットを設定します。 |
| `/keybindings` | - | キーバインド設定ファイルを開く、または作成します。 |
| `/memory` | `status`, `log`, `edit`, `add`, `remove`, `replace`, `write` | CyberCode の長期記憶と指示ファイルを管理します。 |
| `/skills` | - | 利用可能な Skills を表示します。 |
| `/agents` | - | カスタム Agent 設定を管理します。 |
| `/mcp` | `[enable または disable <server>]` | MCP サーバーとツールを管理します。 |
| `/plugin` | `/plugins`, `/marketplace` | プラグインのインストール、有効化、無効化、更新、管理を行います。 |
| `/reload-plugins` | - | 現在のセッションに保留中のプラグイン変更を反映します。 |
| `/hooks` | - | ツールイベント Hook の設定を表示します。 |
| `/ide` | `[open]` | IDE 連携を管理し、状態を表示します。 |
| `/doctor` | - | インストール、設定、実行環境を診断します。 |
| `/login` | - | Anthropic アカウントでログイン、または切り替えます。 |
| `/logout` | - | Anthropic アカウントからログアウトします。 |
| `/release-notes` | - | リリースノートを表示します。 |
| `/feedback` | `/bug [本文]` | フィードバックまたは不具合報告を送信します。 |

## 条件付きで表示されるコマンド

これらはプラットフォーム、アカウント種別、機能フラグ、ポリシーが許可した場合のみ表示されます。

| コマンド | 役割 |

| `/mobile` | モバイルアプリのダウンロード QR コードを表示します。 |
| `/chrome` | Claude in Chrome 連携を設定します。 |
| `/advisor` | 補助 Advisor モデルを設定します。 |
| `/install-github-app` | リポジトリに GitHub Actions 連携を設定します。 |
| `/install-slack-app` | Slack アプリ連携をインストールします。 |
| `/privacy-settings` | プライバシー設定を表示、更新します。 |
| `/stats` | 使用統計とアクティビティを表示します。 |
| `/usage` | プランの使用上限を表示します。 |
| `/extra-usage` | 上限到達時の追加使用量を設定します。 |
| `/upgrade` | より高い上限へのアップグレード入口を表示します。 |
| `/remote-env` | リモートセッションの既定環境を設定します。 |
| `/remote-control` | ローカルターミナルをリモート制御セッションへ接続します。別名は `/rc` です。 |
| `/web-setup` | Web ベースのリモートセッション機能を設定します。 |
| `/session` | リモートセッション URL と QR コードを表示します。 |
| `/voice` | 音声モードを切り替えます。 |

| `/passes` | Claude Code のトライアル権を共有します。 |
| `/stickers` | ステッカー申請フローを開きます。 |

## デスクトップ入力欄のショートカット

デスクトップ入力欄には次の固定コマンドがあります。ローカルパネル、設定、モデルのコマンドはフロントエンドが直接処理し、それ以外は現在の Agent セッションへ送信されます。

| コマンド | 役割 |
|----------|------|
| `/mcp` | 現在のチャットで使える MCP ツールを開きます。 |
| `/skills` | Skills ブラウザーを開きます。 |
| `/help` | デスクトップと Agent のコマンドヘルプを表示します。 |
| `/status` | セッション状態、使用量、コンテキストを表示します。 |
| `/cost` | セッションの使用量とコストを表示します。 |
| `/context` | 現在のコンテキスト使用量を表示します。 |
| `/doctor` | デスクトップ診断を開きます。 |
| `/memory` | このセッションの記憶ファイルを確認します。 |
| `/bug` | フィードバックと不具合報告の入口を開きます。 |
| `/plugin` | プラグイン設定を開きます。 |
| `/config` | デスクトップ設定を開きます。 |
| `/permissions` | 権限設定を開きます。 |
| `/terminal-setup` | ターミナル連携設定を開きます。 |
| `/login` | プロバイダーとアカウントのログイン設定を開きます。 |
| `/logout` | アカウントのログアウト設定を開きます。 |
| `/agents` | Agent 設定を開きます。 |
| `/compact` | 会話コンテキストを圧縮します。 |
| `/clear` | 会話履歴を消去します。 |
| `/review` | コードレビュータスクを開始します。 |
| `/commit` | Git commit を作成します。 |
| `/pr` | Pull Request を作成します。 |
| `/init` | プロジェクト `CYBER.md` を初期化します。 |
| `/model` | モデル切り替えを開きます。 |

デスクトップの別名: `/plugins` は `/plugin`、`/feedback` は `/bug` と同じです。

## 動的コマンド

以下は固定ではなく、プロジェクト、プラグイン、MCP 設定によって変わります。

| 由来 | 形式 | 説明 |
|------|------|------|
| Project Skills | `/skill-name` | ユーザーまたはプロジェクトディレクトリから読み込まれる Skill。 |
| Plugin Skills | `/plugin-skill` | インストール済みプラグインが提供する Skill。 |
| MCP Skills | `/mcp-skill` | MCP サーバーが公開する Prompt または Skill。 |
| Workflow | `/workflow-name` | ワークフロースクリプトから生成されるコマンド。 |

このページとアプリ表示が違う場合は、現在のセッションの `/help`、`/skills`、および `/` 入力後の候補を優先してください。

## 通常 shell の対応コマンド

これらはスラッシュコマンドではありません。TUI に入らず、スクリプトや管理作業で使います。

| Shell コマンド | 役割 |
|----------------|------|
| `cybercode mcp list` | MCP サーバーを一覧表示します。 |
| `cybercode mcp add ...` | MCP サーバーを追加します。 |
| `cybercode mcp remove <name>` | MCP サーバーを削除します。 |
| `cybercode plugin list` | インストール済みプラグインを一覧表示します。 |
| `cybercode plugin install <plugin>` | プラグインをインストールします。 |
| `cybercode plugin uninstall <plugin>` | プラグインをアンインストールします。 |
| `cybercode plugin marketplace list` | プラグインマーケットプレイスを一覧表示します。 |
| `cybercode agents` | カスタム Agent を一覧表示します。 |
| `cybercode doctor` | 実行環境とアップデーター状態を確認します。 |
| `cybercode auth login` | Anthropic アカウントにログインします。 |
| `cybercode auth status` | ログイン状態を表示します。 |
| `cybercode auth logout` | ログアウトします。 |
| `cybercode update` | CLI 更新を確認してインストールします。 |
| `cybercode --help` | インストール済みバージョンの完全な shell コマンドを表示します。 |
