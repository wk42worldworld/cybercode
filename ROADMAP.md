# CyberCode Roadmap

This roadmap is intentionally practical: it focuses on improvements that make CyberCode easier to try, trust, and contribute to.

## Near Term

| Area | Goal | Help wanted |
|------|------|------|
| First-run experience | Make desktop install -> provider setup -> first session feel obvious | Screenshots, short videos, install notes |
| Provider ecosystem | Add more tested provider presets and context-window defaults | MiniMax, OpenRouter, Ollama, DeepSeek, Kimi, local models |
| Documentation | Keep README and module tutorials concise but complete | English, Chinese, Japanese, Korean polish |
| Desktop UX | Smooth session, pending input, provider, and permission flows | Repro steps, UI screenshots, focused PRs |
| Reliability | Improve tests around provider config, sessions, pending input, and scheduled tasks | Vitest coverage and small regression cases |

## Mid Term

| Area | Goal |
|------|------|
| Demo assets | Add short GIF/video demos for desktop setup, provider setup, IM remote control, and scheduled tasks |
| Community | Label beginner-friendly issues and keep contribution paths clear |
| IM adapters | Make Telegram and Feishu setup easier to understand and diagnose |
| Computer Use | Improve platform-specific troubleshooting docs and safety guidance |
| Release notes | Keep each desktop release clear, user-facing, and easy to share |

## Later

| Area | Goal |
|------|------|
| Plugin and Skills ecosystem | More examples and reusable workflows |
| Multi-agent workflows | Better examples for exploration, planning, verification, and teams |
| Local model workflows | More validated Ollama and proxy configurations |
| International docs | More complete Japanese and Korean documentation beyond the README |

## Good First Issue Ideas

- Add a provider preset and document one working model.
- Record a 30-60 second setup demo.
- Improve one README section in a non-English language.
- Add screenshots to a desktop doc page.
- Add a test for one small desktop interaction.
- Reproduce and document one Windows or Linux install issue.
